# Agentic Engineer Course Presentation Constitution

## Core Principles

### I. Content-First Development
Every presentation slide starts with a complete content specification before technical implementation. Specifications must include learning objectives, slide narratives, audience interactions, and success criteria. No slides are created without approved content that defines clear, measurable educational outcomes.

**Rationale**: Ensures pedagogical effectiveness, maintains audience engagement, and enables proper content planning for developer-focused presentations.

### II. Speckit-Slidev Integration
All presentation content must be managed through Speckit framework (.specify/) while leveraging Slidev's developer-focused presentation capabilities. Content specifications drive slide generation, ensuring consistency between educational goals and technical delivery.

**Rationale**: Combines structured specification management with modern developer presentation tools for maintainable, version-controlled educational content.

### III. Developer-Focused Pedagogy (NON-NEGOTIABLE)
All content must be tailored for software developers learning agentic engineering concepts. Code examples must be executable, concepts must include practical implementations, and learning materials must follow hands-on methodology with demos and interactive elements.

**Rationale**: Matches audience expertise level and learning preferences, ensuring maximum knowledge transfer and practical skill development.

### IV. Specification-Driven Slides
Every slide section requires structured documentation: content specifications (learning objectives, key concepts), technical examples (code snippets, demos), and assessment criteria (quiz questions, exercises). Slide content must be derived from approved specifications.

**Rationale**: Maintains educational quality standards, enables content reuse and updates, and ensures comprehensive coverage of course objectives.

### V. Modular Content Architecture
Presentation content must be organized into independent modules with clear dependencies. Each module should deliver standalone educational value and be presentable as a self-contained learning unit without requiring other modules.

**Rationale**: Enables flexible course delivery, supports different learning paths, and allows content reuse across different presentation contexts.

## Presentation Workflow

All presentation development follows this structured workflow:

1. **Content Specification Phase**: Define learning objectives, target audience, and key concepts for each module
2. **Technical Planning Phase**: Identify code examples, demos, and interactive elements needed
3. **Slide Design Phase**: Create Slidev-compatible markdown with proper theming and navigation
4. **Content Integration Phase**: Combine Speckit specifications with Slidev presentation framework
5. **Review Phase**: Validate educational objectives and technical accuracy
6. **Delivery Preparation**: Test presentation flow, demos, and audience interaction elements

**Amendment Procedure**: This workflow ensures educational quality. Skipping phases requires explicit justification in presentation plan documentation.

## Content Organization

### Educational Structure
- Clear learning objectives for each presentation module
- Progressive skill building from basic to advanced concepts
- Practical examples relevant to agentic engineering
- Assessment checkpoints to validate understanding

### Technical Integration
- Executable code examples that demonstrate concepts
- Live demos showing agentic development workflows
- Interactive elements leveraging Slidev's capabilities
- Version-controlled content enabling iterative improvements

### Audience Engagement
- Developer-appropriate technical depth and terminology
- Hands-on exercises and coding challenges
- Real-world case studies from agentic engineering projects
- Q&A sections addressing common implementation questions

## Quality Assurance

### Content Quality Gates
- All concepts must include practical implementation examples
- Code samples must be tested and executable
- Educational objectives must be measurable and assessable
- Presentation flow must be validated for timing and pacing

### Technical Standards
- Slidev markdown must follow best practices for readability
- Animations and transitions must enhance rather than distract from content
- Code syntax highlighting and formatting must be consistent
- Export formats must maintain visual quality across different viewing platforms

### Educational Effectiveness
Learning outcomes must be validated through audience feedback and comprehension assessments. Content updates require review of educational impact and alignment with course objectives.

## Governance

This constitution governs presentation content creation and delivery for the Agentic Engineer Course. All content specifications and slide development must verify compliance through mandatory Constitution Check sections in planning documentation.

**Amendment Process**: Constitution changes require version increment (MAJOR for principle changes, MINOR for new sections, PATCH for clarifications) with documented educational impact assessment.

**Content Review**: All presentation modules must validate adherence to educational principles. Deviations require explicit justification in content specifications.

**Technical Compliance**: Speckit specifications must align with Slidev capabilities. Technical limitations must be documented and addressed through alternative presentation methods.
