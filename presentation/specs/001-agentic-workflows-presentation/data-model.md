# Data Model: Agentic Workflows Presentation

**Date**: 2026-04-16 | **Feature**: 001-agentic-workflows-presentation

## Overview

This presentation project has no runtime data model (no database, API, or persistent state). The "entities" below describe the **content model** — the logical structure of presentation artifacts and their relationships.

## Content Entities

### Slide

The atomic unit of presentation content.

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique slide identifier (auto-generated by Slidev from order) |
| layout | enum | Slidev layout: `cover`, `default`, `two-cols`, `image-left`, `section`, `end` |
| section | ref → Section | Parent section this slide belongs to |
| title | string | Slide heading (optional for some layouts) |
| content | markdown | Slide body content in Slidev-flavored markdown |
| notes | markdown | Presenter notes (visible only in presenter mode) |
| transition | string | Slide transition effect (optional) |
| clicks | number | Number of click-through animations on this slide |

### Section

A logical grouping of slides forming one part of the presentation arc.

| Field | Type | Description |
|-------|------|-------------|
| name | string | Section identifier matching FR-001 structure |
| order | number | Position in presentation flow (1-6) |
| duration_min | number | Target duration in minutes |
| learning_objective | string | What attendees should understand after this section |
| slides | ref[] → Slide | Ordered list of slides in this section |

**Sections (from FR-001)**:
1. What Are Agentic Workflows (8 min)
2. Why They Matter (7 min)
3. Live Claude Code Demo (15 min)
4. Workflow Patterns (10 min)
5. Practical Tips (8 min)
6. Q&A (7 min)

### CodeExample

An executable code snippet demonstrated during the presentation.

| Field | Type | Description |
|-------|------|-------------|
| name | string | Descriptive name for the example |
| file_path | string | Path in `slides/snippets/` or `examples/` directory |
| language | string | Programming language for syntax highlighting |
| section | ref → Section | Section where this example is shown |
| is_live_demo | boolean | Whether this is part of the live Claude Code demo |
| description | string | What this example demonstrates |
| prerequisites | string | Setup needed before executing (if any) |

### ThemeConfig

Visual styling configuration matching the referenced Google Slides theme.

| Field | Type | Description |
|-------|------|-------------|
| background_color | string | Primary slide background color |
| text_color | string | Primary text color |
| accent_color | string | Accent/highlight color |
| heading_font | string | Font family for headings |
| body_font | string | Font family for body text |
| code_font | string | Font family for code blocks |
| code_theme | string | Shiki syntax highlighting theme |

**Note**: Exact values pending theme extraction from referenced Google Slides (see research.md R2).

## Relationships

```text
Section 1──* Slide
Section 1──* CodeExample
Slide *──* CodeExample (a slide can reference multiple examples; an example can appear on multiple slides)
ThemeConfig 1──1 Presentation (global configuration)
```

## State Transitions

Not applicable — presentation content is static. Slides advance linearly with optional click-through animations within individual slides.

### LearningObjective

Educational goals tracked for assessment and constitutional compliance.

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique identifier |
| section | ref → Section | Parent section |
| objective | string | Action-oriented statement (starts with verb) |
| bloom_level | enum | Cognitive level: remember, understand, apply, analyze, evaluate, create |
| assessment_method | enum | quiz, observation, survey |
| success_criteria | string | Measurable outcome (maps to SC-001 through SC-005) |

**Examples**:
- "Define agentic workflows and explain key differences from traditional development" (understand, quiz, SC-001)
- "Identify 3+ practical use cases in their work" (apply, observation, SC-002)

### DemoSegment

Live coding demonstration script with rehearsal tracking.

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique identifier (e.g., "demo-01-basics") |
| title | string | Segment title |
| order | number | Position within demo (1-3) |
| duration_min | number | Target time (typically 3 minutes) |
| script | markdown | Step-by-step instructions for presenter |
| code_examples | ref[] → CodeExample | Examples demonstrated |
| validation_checkpoints | string[] | Expected outcomes at each step |
| backup_recording | string | Path to MP4 in `/public/demos/` |
| rehearsal_status | enum | scripted, rehearsed, validated |

**Rehearsal States**:
```
scripted → rehearsed (after practice run) → validated (successful execution)
```

### Asset

Presentation media assets with optimization tracking.

| Field | Type | Description |
|-------|------|-------------|
| file_path | string | Path in `/public/` (screenshots/, diagrams/, demos/) |
| type | enum | screenshot, diagram, video |
| format | string | png, svg, mp4 |
| file_size_kb | number | Actual file size |
| alt_text | string | Accessibility description |

**Optimization Requirements**:
- Screenshots/diagrams: ≤200 KB
- Videos: ≤60 seconds, H.264 codec, 2-5 Mbps
- All assets must have alt_text

### AssessmentItem

Quiz questions and evaluation criteria.

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique identifier |
| section | ref → Section | Parent section |
| question | string | Question text or observation criteria |
| type | enum | multiple_choice, short_answer, observation |
| correct_answer | string | For quiz types (optional) |
| success_threshold | string | Target % (aligns with SC-001 through SC-005) |

order: 1

- Define agentic workflows (understand, quiz, SC-001)

## Validation Rules

**Content Completeness**:
- Every Section must have ≥2 LearningObjectives
- Every Section must have ≥1 AssessmentItem
- Every Section must have at least 1 Slide
- Sum of all section durations must be 55-60 minutes

**Code Quality**:
- Every CodeExample with `is_live_demo: true` must be tested and executable
- Code blocks displayed in slides must be ≤20 lines
- All code examples must exist in repository

**Educational Quality**:
- LearningObjectives must use action verbs
- Success criteria must be measurable and map to SC-001 through SC-005
- Sections must cover at least 2 Bloom taxonomy levels

**Asset Quality**:
- All Asset file paths must exist
- File sizes must meet optimization requirements
- All assets must have alt_text for accessibility

**Demo Readiness**:
- All DemoSegments must have `rehearsal_status: validated` before presentation
- Total demo duration must be ≤15 minutes
- All demo segments must have backup recordings

**Slide Density**:
- Maximum 50 words body text per slide (concept slides)
- Maximum 6 elements per slide (bullets, images, code blocks)
- Font sizes ≥18pt for readability

## State Tracking

```
draft → (content review) → reviewed → (approval) → approved → (generation) → generated
```

### Demo Rehearsal Workflow
```
scripted → (practice) → rehearsed → (validation) → validated
```

### Code Example Validation
```
untested → (execution) → passing | failing
```
